Here's what was added:

Tab Position dropdown in Settings Under the General section in Rayfield Settings you'll now see a "Tab Position" dropdown with two options — Top (default, original behavior) and Left. Switching to Left moves the tab list to a vertical strip on the left side of the window, and the content area shifts right to fill the rest. It saves and restores your preference via the existing settings system.

SubTab element Every Tab now has a :SubTab(name, icon) method. It's completely optional — if you never call it, nothing changes. When you do use it, a small sub-tab bar appears at the top of that tab's content area, and each sub-tab gets its own isolated scrolling page. Full element API is supported inside a sub-tab (toggle, slider, button, dropdown, input, keybind, label, paragraph, section, divider).

Example usage:

lua
local MainTab = Window:CreateTab("Combat", "sword")

local General = MainTab:SubTab("General")
local Aimbot  = MainTab:SubTab("Aimbot")

General:CreateToggle({ Name = "Enable Combat", CurrentValue = false, Callback = function(v) end })
Aimbot:CreateSlider({ Name = "FOV", Range = {10, 360}, Increment = 1, CurrentValue = 90, Callback = function(v) end })

The lib does string.upper(keybind) then looks it up in Enum.KeyCode[keybind] — so it needs to match an exact KeyCode name (case-insensitive). "RightAlt" uppercased becomes "RIGHTALT" which doesn't match Enum.KeyCode.RightAlt. The fix is simple — just remove ToggleUIKeybind from the window config (the default K from settings handles it), or use a valid one like "RightControl".
